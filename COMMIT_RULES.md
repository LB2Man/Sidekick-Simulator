# Commit Rules

Every commit message must use one of these formats:

- `Fix | Description` — bug fixes and corrections.
- `Feat | Description` — new features and enhancements.
- `Test | Description` — test additions and test-only changes.

Write the description in imperative form and keep it concise.

Examples:

- `Fix | Align HUD panels along the top edge`
- `Feat | Add a collapsible task panel`
- `Test | Cover task completion rewards`
